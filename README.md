# try-cursor-pub

## Cursor 当前能力概览

[Cursor](https://www.cursor.com/) 是一款基于 AI 的智能代码编辑器，在 VS Code 基础上深度集成了大语言模型能力。以下是其核心功能：

### 智能代码编辑

- **Tab 自动补全**：基于上下文的多行代码智能补全，能够预测下一步编辑意图，支持跨行、跨块的连续建议。
- **Inline Edit（Cmd/Ctrl+K）**：在编辑器内通过自然语言指令直接对选中代码进行修改、重构或生成，无需离开代码上下文。
- **智能重写**：检测到手动编辑后，自动建议对相关代码进行一致性修改。

### 对话式 AI 助手

- **Ask 模式**：只读模式，用于探索代码库、提问和理解代码逻辑，不会对文件做任何修改。
- **Agent 模式（默认）**：全自动编码代理，可以自主搜索代码库、读取文档、编辑多个文件、执行终端命令、并自动修复 lint/类型错误，端到端完成复杂任务。
- **Plan 模式**：先制定实施计划再编码，适合需要谨慎设计的大型变更。
- **Debug 模式**：系统化的故障排查模式，通过收集运行时证据定位 bug。
- **自定义模式**：用户可以自定义工具集和系统提示词，创建专用的工作流模式。

### 上下文管理（@ 符号）

通过 `@` 符号可以精确地为 AI 提供上下文：

- `@Files` / `@Folders`：引用特定文件或文件夹
- `@Code`：引用具体的函数、类等代码符号
- `@Docs`：引用第三方库文档（支持自动抓取和索引）
- `@Web`：实时搜索网络获取最新信息
- `@Git`：引用 Git 历史、diff、PR 等信息
- `@Codebase`：对整个代码库进行语义检索

### Cloud Agent（云端代理）

- 在云端沙箱中自主运行，可以在后台处理复杂的多步骤任务。
- 支持从 GitHub Issue 或 PR 评论直接触发。
- 具备完整的工具链：代码搜索、文件编辑、终端执行、Git 操作、创建 PR 等。
- 可并行启动多个代理处理不同任务。

### 其他能力

- **Rules 系统**：通过项目级别的 `.cursor/rules` 文件定义编码规范、架构约束和 AI 行为偏好，确保 AI 输出与团队风格一致。
- **MCP（Model Context Protocol）支持**：可连接外部工具和数据源（如数据库、API、设计工具等），扩展 AI 的能力边界。
- **多模型支持**：内置支持 Claude、GPT、Gemini 等多种前沿模型，用户可按需选择。
- **隐私模式**：开启后代码不会被存储或用于训练，适合敏感项目。
- **代码库索引**：自动为项目建立语义索引，提升代码搜索和上下文理解的准确性。
